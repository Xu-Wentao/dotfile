
# PPT 内容规划指南（Content Planning Guide）

> **MANDATORY**: 在开始创建或修改任何PPT之前，必须先完成内容规划。直接动手生成PPT而不做规划，是导致内容混乱、信息丢失、排版灾难的根因。

## 核心原则

1. **先规划，后生成** — 永远不要拿到需求就直接写代码/生成PPT
2. **结构化优先** — 所有内容必须先整理成结构化的数据格式（表格、JSON、大纲等）
3. **适配受众** — 根据目标受众调整内容深度和表达方式
4. **一页一主题** — 每张幻灯片只传达一个核心信息

---

## 第一步：需求分析与内容结构化

### 从用户输入中提取结构化数据

用户通常以自然语言描述需求，其中隐含了结构化信息。你必须先将其提取为结构化数据。

**示例 — 用户给了一段任务清单：**
> 完成6项任务，执行人，各支部；时间1月5日-1月9日...

**错误做法**：直接把这段文字塞进PPT → 导致文字截断、信息丢失、结构混乱

**正确做法**：先提取为结构化表格：

```markdown
| 序号 | 任务名称 | 执行人 | 开始日期 | 结束日期 | 阶段 |
|------|---------|--------|----------|----------|------|
| 1    | 完成集团党建工作满意度测评 | 各支部 | 1月5日 | 1月9日 | 筹备期 |
| 2    | 完成党费基数核算及党费补缴方案 | — | 1月7日 | 1月9日 | 筹备期 |
| ...  | ... | ... | ... | ... | ... |
```

### 结构化检查清单

在动手前，确认以下信息都已提取完整：

- [ ] **所有条目**都已列出，没有遗漏
- [ ] **每条信息**的字段完整（名称、时间、负责人、分类等）
- [ ] **时间线**是否有重叠或衔接
- [ ] **分类/阶段**是否合理
- [ ] 任务名称是否**精简但完整**（不要截断，也不要过于冗长）

---

## 第二步：内容提炼与信息分层

### 原则：提炼要点，而非搬运原文

当从PDF、文档等外部源提取内容时，**绝对不要逐字搬运**。需要经过以下处理：

### 信息分层模型

```
原始内容（PDF/文档/资料）
    ↓ 提炼
核心论点（每页1-2个）
    ↓ 精简
幻灯片文字（每条 ≤ 25字）
    ↓ 可视化
图表/图示/关键词
```

### 每页幻灯片内容限制

| 元素 | 上限 | 说明 |
|------|------|------|
| 核心论点 | 1-2个 | 每页只讲一件事 |
| 条目/要点 | 3-5个 | 超过5个考虑拆页 |
| 每条文字 | ≤ 25字 | 超过需精简或用图示 |
| 一句话概念 | ≤ 15字 | 标题/标签级别 |
| 总文字量 | ≤ 80字 | 单页所有文字总和 |

### 内容精简技巧

1. **去除冗余**：删除"随着...的发展"、"在...背景下"等套话
2. **名词化**："Agent可以进行自主的规划和决策" → "Agent核心能力：规划与决策"
3. **数据化**："性能大幅提升" → "推理速度提升3倍"
4. **层次化**：把长段落拆分为"标题 + 3个关键词"
5. **对比化**："A比B好" → 用对比表格/图表呈现

### 受众适配

| 受众类型 | 内容策略 | 语言风格 |
|---------|---------|---------|
| 技术专家 | 深入细节，保留术语 | 专业、精确 |
| 技术小白 | 通俗类比，多用图示 | 通俗、生动 |
| 管理层 | 聚焦结论和数据 | 简洁、结果导向 |
| 综合受众 | 兼顾深度与广度 | 分层呈现 |

## 第三步：页面规划与大纲设计

### 页面大纲模板

在生成PPT前，先写出完整的大纲，包含每页的：

```markdown
## 页面大纲

### 第1页：封面
- 标题：XXX
- 副标题：XXX
- 汇报人/日期

### 第2页：目录/概览
- 列出3-5个章节

### 第3页：章节一标题
- 要点1（≤25字）
- 要点2
- 要点3
- [配图/图表说明]

### 第4页：章节一详情
- 数据/图表展示
- 核心结论

...

### 布局规划提示（每页大纲中应标注）
- 页面类型：单栏 / 双栏 / 三栏 / 四栏
- 封面/结束页：内容必须水平居中
- 卡片/内容区的目标高度（应填充内容区可用高度的70-90%）
- 如果双栏布局：左右栏内容量是否均衡？否则需调整
```

### 页面数量参考

| 内容类型 | 建议页数 | 说明 |
|---------|---------|------|
| 5分钟分享 | 5-8页 | 封面+目录+3-5内容页+结束 |
| 15分钟分享 | 10-15页 | 封面+目录+8-12内容页+结束 |
| 30分钟汇报 | 20-30页 | 可包含详细数据页 |
| 1小时培训 | 40-60页 | 含练习、案例页 |

## 第四步：内容-模板映射

### 评估内容与模板的匹配度

在决定使用哪个模板页/布局前，先评估：

1. **内容密度**：该页要放多少文字/图表？
   - 低密度（1-3个要点）→ 标题+要点布局
   - 中密度（4-6个要点+图表）→ 双栏布局
   - 高密度（表格、时间线、数据密集）→ 全幅布局或专用模板

2. **内容类型**：
   - 叙事/流程 → 左文右图/流程图布局
   - 对比/并列 → 多栏布局
   - 数据展示 → 图表布局
   - 时间线 → 专用时间线/甘特图模板
   - 分类/总结 → 矩阵/表格布局

3. **视觉重点**：
   - 文字为主 → 确保足够的文本区域
   - 图表为主 → 留出大面积图表区域
   - 混合型 → 双栏或图文叠加

### 文字截断预防

**CRITICAL**: 在向表格单元格、文本框等有限空间填充内容时：

1. **先测量，再填充**：使用 inventory.py 获取每个 shape 的尺寸
2. **计算可用字符数**：
   ```
   可用字符数 ≈ (shape宽度英寸 × 96 / 字号pt) × 行数
   ```
3. **超长文字处理策略**（按优先级）：
   - 精简文字内容（首选）
   - 缩小字号（下限不低于8pt）
   - 拆分到多个shape/单元格
   - 拆分到多页
4. **绝对禁止**：让文字超出shape边界导致截断

## 第五步：质量自查清单

在最终输出前，对照以下清单检查：

### 内容完整性
- [ ] 用户要求的所有信息都已包含
- [ ] 没有文字被截断（特别是表格单元格中的长文本）
- [ ] 任务/条目名称完整（没有被截断为"完成党支部履职文"这种情况）
- [ ] 时间/数据等关键信息准确无误
- [ ] 每一项任务/条目都有完整名称出现在PPT中某处（不依赖shape与表格的"隐式对应"）

### 结构逻辑
- [ ] 页面顺序符合逻辑（先总后分、先背景后结论）
- [ ] 每页只传达一个核心信息
- [ ] 信息层次清晰（标题→要点→细节）
- [ ] 相邻页面之间有逻辑衔接

### 排版质量
- [ ] 文字没有溢出边界
- [ ] 元素之间没有重叠/遮挡
- [ ] 字号层次合理（标题 > 正文 > 注释）
- [ ] 颜色对比度足够（文字可读）
- [ ] 对齐一致（同类元素对齐方式统一）
- [ ] 修改后的内容保持了原模板的配色和排版风格一致
- [ ] 内容区空间利用率 ≥ 70%（卡片/内容填充了标题和底栏之间的大部分空间，非<50%）
- [ ] 页面布局平衡（左右两侧分布合理，无单侧大面积空白）
- [ ] 封面/结束页核心内容水平居中
- [ ] 结构性标题视觉权重 > 示例/比喻等辅助文字
- [ ] 同级元素在不同栏中垂直对齐
- [ ] 并列卡片/列表项高度基本一致

### 品牌元素复用（参考模板场景 MANDATORY）
- [ ] Logo/公司标识已从参考模板提取并在每页复用
- [ ] Logo位置和尺寸与参考模板一致（左上/右上/居中等）
- [ ] 封面页背景图与参考模板一致
- [ ] 标题栏样式（颜色、位置、字号、是否有背景矩形）与参考模板一致
- [ ] 页脚/底部栏与参考模板一致
- [ ] 装饰元素（线条、分隔符）与参考模板一致
- [ ] 整体视觉感"看起来像同一系列的作品"
- [ ] 没有引入参考模板中不存在的配色或装饰元素

### 受众适配
- [ ] 语言风格适合目标受众
- [ ] 技术深度适中
- [ ] 术语使用恰当（小白受众有解释）

---

## 编辑现有PPT的特殊检查项

当编辑（而非新建）现有PPT时，除了上述通用清单外，还需额外检查：

### 修改前后一致性
- [ ] 原模板中所有用户可见的文字都被正确替换（没有遗漏的旧内容）
- [ ] 品牌信息、页码、水印等辅助元素已按需更新
- [ ] 没有残留的占位符文字（如"Lorem ipsum"、示例内容等）

---

## 参考模板创建新PPT的特殊规划要求

当任务是"参考某PPT的形式/风格，基于外部内容创建新PPT"时，除了上述通用规划步骤外，**必须在内容规划阶段完成品牌元素的提取和规划**。

### 品牌元素提取步骤（在内容规划阶段完成，不是在写代码阶段）

1. **Unpack参考模板**，列出所有媒体文件：
    ```bash
    python ooxml/scripts/unpack.py reference.pptx reference-unpacked
    ls reference-unpacked/ppt/media/
    ```

2. **识别品牌图片（CRITICAL - 必须同时检查slide、slideLayout、slideMaster三层）**：
    - Logo图片（通过XML中所有 `r:embed` 引用定位）
    - **背景图**（在 `slideLayout` 的 `<p:bg>` 元素中，**不一定在slide XML里**）
    - 其他品牌装饰图片
    ```bash
    # 检查slideLayout中的背景图（最常被遗漏的步骤）
    grep -rn "<p:bg>" reference-unpacked/ppt/slideLayouts/
    grep -rn "r:embed" reference-unpacked/ppt/slideLayouts/
    grep -rn "duotone" reference-unpacked/ppt/slideLayouts/
    ```

3. **记录每个品牌图片的精确位置**：
    - 从slide XML、slideLayout XML中提取每个图片shape的 `<a:off x="..." y="..."/>` 和 `<a:ext cx="..." cy="..."/>`
    - 将EMU转换为英寸（÷914400），记录在design-spec.md中
    - **标注图片来源**：是来自slide、slideLayout还是slideMaster

4. **规划品牌元素的复用方案**：
    ```markdown
    ## 品牌元素复用规划
    
    | 页面类型 | Logo位置 | 背景图 | 页脚栏 | 其他装饰 |
    |---------|---------|--------|--------|---------|
    | 封面页  | 左上+右上 | 全幅背景图(slideLayout,需调色) | 无(模板无) | 无(模板无) |
    | 内容页  | 右上角   | slideLayout背景(需调色) | 无(模板无) | 模板特有装饰 |
    | 结束页  | 右上角   | 全幅背景图(slideLayout,需调色) | 无(模板无) | 无(模板无) |
    ```
    **注意**：模板中没有的元素（如底部栏、分隔线）**不要**自行添加。

### 配色切换时的背景图处理（MANDATORY）

当需要将模板配色从A色切换到B色时：

1. **提取原始背景图**（从slideLayout的 `<p:bg>` 中定位引用的媒体文件）
2. **记录色调处理参数**（如duotone的shade值、satMod值）
3. **用Sharp重新生成调色版背景图**，而非丢弃背景图改为纯色
4. **绝对禁止**：发现模板有背景图但用 `#XXXXXX` 纯色替代

### 设计规范完整性检查

在开始创建PPT之前，确认design-spec.md包含以下**所有**信息：

- [ ] **品牌元素**：Logo图片路径、每页Logo位置和尺寸、背景图路径及来源(slide/layout/master)
- [ ] **配色**：主色、辅色、背景色、卡片色、文字色（全部有hex值）
- [ ] **字体**：标题字体+字号+对齐方式、正文字体+字号、注释字体+字号
- [ ] **标题栏**：精确位置、尺寸、颜色（是红字白底还是白字红底？有无背景矩形？）
- [ ] **卡片/面板**：圆角大小、边框颜色和粗细、背景色、标签样式
- [ ] **页脚/底部栏**：位置、尺寸、背景色
- [ ] **不要自行新增的元素**：模板中没有的装饰色、装饰形状不应出现在新PPT中

### 表格/数据密集型编辑
- [ ] 表格行列数未发生非预期的变更
- [ ] 每个单元格的文字完整、无截断
- [ ] 合并单元格的结构保持不变（除非用户明确要求）
- [ ] 列宽比例保持不变（除非需要适应新内容）
- [ ] 单元格内的文字大小和颜色与同列其他单元格一致

### 浮动元素对齐
- [ ] 浮动shape（如甘特条形）的位置与表格行/列正确对齐
- [ ] shape的宽度与其表示的时间跨度/数据范围匹配
- [ ] 多个浮动shape之间没有重叠
- [ ] 修改后的shape没有超出幻灯片边界

### 甘特图编辑专项检查
- [ ] 原始表格结构已完整记录（行数、列数、每列含义、列宽）
- [ ] 时间轴映射已规划（新内容的每个任务对应哪一行、哪几列）
- [ ] 列的时间粒度与原模板保持一致（按天不改为按周）
- [ ] 日期行正确反映了新内容的时间范围
- [ ] 没有空行浪费空间（如果有，检查是否任务分布不均）
- [ ] 表格单元格内没有冗余的阶段标签（阶段信息应只出现在专用标签位置）
- [ ] 甘特条形的文字已充分精简（不超过条形可容纳字符数）
- [ ] 甘特条形的位置(y)与对应表格行对齐
- [ ] 甘特条形的宽度(width)与任务的时间跨度匹配
- [ ] 原模板中的编号、旧品牌信息等已清理
- [ ] 每个任务名称在PPT中有完整的文字呈现（不依赖隐式对应）

### 视觉验证（MANDATORY for complex edits）
- [ ] 使用 `inventory.py` 检查所有shape的文字是否完整、无overflow
- [ ] 如有可能，生成缩略图与原模板对比，确认风格一致
- [ ] 特别检查group shape内的元素（inventory可能无法检测到group内的overflow）
