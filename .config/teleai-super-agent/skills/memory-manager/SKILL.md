# Memory Manager

跨会话长期记忆：用户指定（USER.md）与系统推断（MEMORY.md）会载入系统提示；每日日志仅通过 memory_search / memory_get 按需检索。

**记忆根目录（本项目约定）**：`~/.local/share/teleai-super-agent/memory/`
- USER.md：用户指定的长期记忆
- MEMORY.md：系统推断的核心事实记忆
- 当日日志：`daily-log/YYYY-MM-DD.md`（日期按 YYYY-MM-DD）

在对话中，如需引用具体文件，请使用上述路径进行描述。

---

## 一、用户指定长期记忆（USER.md）

**用途**：存放**由用户明确指定**的、固定的、长期的记忆，例如：
- 用户长期偏好（语言、输出风格、工作习惯等）
- 身份/角色信息（职业、常用工具、项目背景等）
- 需要「每次对话默认遵守」的设定

**USER.md 的格式**：用二级标题（##）分大类，每条事实用列表项（-）；不要写成长篇段落，保持精炼。

**何时写入**：仅当用户**明确表达**希望被长期记住时（例如「请记住，我是程序员」「我的职业是xxx」），你需要在回答中帮助用户**整理出应写入 USER.md 的完整内容**，并用自然语言清楚说明：

- 哪些信息应被视为长期事实/偏好；
- 建议的 USER.md 结构（使用二级标题与列表项）；
- 目标文件是 `~/.local/share/teleai-super-agent/memory/USER.md`。

实际的文件读写由底层系统和工具托管完成，你**不要在回答中指定具体工具名或调用格式**，只需给出清晰的文本内容与目标文件说明。

**何时不写**：单次对话中的临时信息、某一天的工作内容、需要按时间衰减的流水记录，应写入「每日日志」，不要写入 USER.md。

---

## 二、推断长期记忆（MEMORY.md）

**用途**：存放**系统根据历史会话推断整理**的长期记忆（偏好/设定/稳定事实）。它会自动维护（定期合并每日日志），但也应遵循"长期稳定、可复用、无时间戳"的原则。

**边界**：
- 若与用户在当前对话中的最新指令冲突，以用户最新指令为准。
- 用户明确指定要"长期记住"的内容应写入 `USER.md`，不要写进 `MEMORY.md`。

---

## 三、每日日志记忆（daily-log/YYYY-MM-DD.md）

**用途**：按日记录的会话摘要与工作小结，按时间衰减、不自动载入上下文，仅通过搜索查询。

**写入时机**：
- **定时任务（隐式）**：系统每 2 小时会触发后台 Agent（`memory-daily-log`）总结最近会话，并将生成的摘要**追加写入**到当日日志文件 `daily-log/YYYY-MM-DD.md`。
- **显式写入（通过 Write 工具）**：当用户明确要求「把今天/这段记一下」「帮我把刚才的讨论记到今天的日志」等时，你可以：
  - 先在对话中对需要记录的内容进行简要总结；
  - 使用 opencode 的 `Write` 等文件写入工具，将这些日志**追加写入**到当日的 `daily-log/YYYY-MM-DD.md` 文件末尾（必要时先用 `Read` 读取原文件，再用 `Write` 以「原内容 + 新内容」的方式覆盖写入，实现逻辑上的"追加"）。

**格式**：每日记忆文件为 Markdown，后台 Agent 会自动生成结构化摘要。显式写入时建议保持简洁明了。

**对「某日记忆」的修改**：若用户调取了某日的每日日志（例如 2026-03-08.md）并对其中的内容进行**修改、订正或补充**，你**不要**直接改写该日的 .md 文件。正确做法是：把「修改/订正后的结论」作为**新的一条记录**追加到**当日的**每日日志中，由后台 Agent 在下一次写入时一并体现。这样既保留历史，又明确更新发生在「今天」。

---

## 四、工具使用流程

| 用户意图 | 操作 |
|----------|------|
| 「记住 X」（长期、固定，且用户明确指定） | 在对话中用自然语言确认/整理后，使用通用 `Write` 文件工具对 `~/.local/share/teleai-super-agent/memory/USER.md` 进行**覆盖写入**（先 Read 再 Write，生成新的完整内容，格式 ## + -）。 |
| 「把今天/这段记一下」 | 对需要记录的内容做简要总结，使用 `Write` 工具**追加**到当日 `daily-log/YYYY-MM-DD.md`（先 Read 原文件，再 Write 原内容+新内容）。 |
| 「查某天/之前说过什么」 | 先 `memory_search(query=...)` 在每日日志中检索；需要某日完整日志时用 `memory_get(file=YYYY-MM-DD.md)` 取出整份文件。 |
| 想回顾/核对某条长期设定 | 读取 `USER.md` / `MEMORY.md`：`memory_get(file=USER.md)` 或 `memory_get(file=MEMORY.md)`。 |

**查询流程**：先用 memory_search 得到匹配的片段与文件名；需要某日完整日志时，用 memory_get(file=文件名) 读取整份文件；需要长期设定时读取 USER.md / MEMORY.md。

---

## 五、长期记忆载入与长度限制

- 应用启动/生成会话时，会将 `USER.md` 与 `MEMORY.md` 的内容**自动载入系统提示词**，因此长期事实在每次对话中默认可见。
- 若任一文件超过约定长度阈值，载入时会**只取前一段并截断**，并在末尾提示「完整内容可通过 memory_get(file=...) 按需查询」。
