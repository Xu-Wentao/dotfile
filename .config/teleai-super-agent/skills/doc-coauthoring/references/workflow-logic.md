# Workflow Logic (工作逻辑)

文档不仅填满「模版小节」，还应**按真实工作顺序**组织内容，便于读者理解与评审。以下为推荐写作/思考顺序。

## 核心原则

- **工作逻辑**：先问题、再方案、再执行、再风险——与读者和评审的认知顺序一致。
- **模版结构**：由「文档类型」决定小节标题（见 internal-templates.md），但**撰写顺序**可按下述逻辑调整。

## 按文档类型的推荐写作顺序

| 类型 | 推荐顺序（按工作逻辑） |
|------|------------------------|
| 技术方案 / 设计文档 | ① 背景与目标 → ② 现状与问题 → ③ 方案设计/技术选型 → ④ 实施计划 → ⑤ 风险与依赖 |
| 需求说明书 / PRD | ① 需求背景与目标用户 → ② 功能/非功能需求 → ③ 验收标准与优先级 |
| 决策文档 / ADR / RFC | ① Context/背景 → ② 方案与 Alternatives → ③ Decision/结论 → ④ Consequences/影响与后续 |
| 评审文档 | ① 评审目的与材料摘要 → ② 结论与决议 → ③ 待办与责任人 |
| 立项书 / 周报 | ① 背景与目标/完成情况 → ② 范围与计划/下周计划 → ③ 风险与需协调 |

## 在技能中的用法

- **Stage 2 - Section ordering**：在「按模版生成小节」之后，提示用户：建议按**工作逻辑**顺序逐节撰写（例如先写背景与问题，再写方案，最后写风险）；若用户同意，则按上表顺序引导填写，必要时可合并或拆分小节以兼顾模版与逻辑。
