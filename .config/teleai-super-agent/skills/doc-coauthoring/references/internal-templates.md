# Internal Document Templates (内部文档模版)

本技能优先使用内部模版以规范技术文档。若用户未指定模版，可从此列表选择或推荐。

## 文档类型与模版对应

### 国内常见类型 (Domestic)

| 文档类型 | 典型小节结构 | 适用场景 |
|---------|-------------|----------|
| 技术方案 | 背景与目标、现状与问题、方案设计、技术选型、实施计划、风险与依赖 | 技术评审、立项、架构决策 |
| 需求说明书 | 需求背景、目标用户、功能需求、非功能需求、验收标准 | 产品需求、需求评审 |
| 评审文档 | 评审目的、材料摘要、结论与决议、待办与责任人 | 技术/方案评审、复盘 |
| 立项书 | 项目背景、目标与范围、资源与排期、风险与预案 | 立项、资源申请 |
| 周报/月报 | 本周完成、下周计划、风险与阻塞、需协调事项 | 汇报、同步 |
| 设计文档 | 背景、设计目标、方案对比、详细设计、接口与数据 | 详细设计、技术设计评审 |

### 西方/通用类型 (Western / General)

| 文档类型 | 典型小节结构 | 适用场景 |
|---------|-------------|----------|
| PRD (Product Requirements) | Problem, Goals, User Stories, Scope, Success Metrics | 产品需求 |
| RFC / Design Doc | Context, Proposal, Alternatives, Open Questions | 技术提案、变更 |
| Decision Doc / ADR | Context, Decision, Consequences | 架构决策记录 |
| Technical Spec | Overview, Approach, Interfaces, Rollout | 技术规格 |

## 使用说明

- **Stage 0 / Stage 1**：优先问用户「是否使用公司内部文档模版？」或「从上述类型中选择一种」；若用户无指定，则根据其描述推荐一种内部类型并对应到上表小节。
- **Stage 2**：一旦确定类型，文档的小节与占位符**严格按上表对应结构**生成，不可随意发明小节名。
