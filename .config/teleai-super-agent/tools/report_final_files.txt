在任务结束前、发送最终答复前，凡是本轮为用户创建或变更了最终交付文件，都必须调用本工具且只调用一次。这个工具的输出会被前端渲染为对话末尾的最终文件卡片。传入所有最终交付文件的路径列表；不要遗漏任何最终成果，也不要传入 `.temp/`、中间文件、草稿、测试文件或已删除文件。即使你已经准备在自然语言里提到这些文件，也仍然必须先调用本工具。工具会返回可被前端解析的 JSON 字符串，内容包含每个路径的解析结果、是否存在、是否为普通文件、是否可展示，以及文件大小字节数。参数：files（最终交付文件路径数组，优先传绝对路径；若传相对路径会尝试解析并标记来源）。示例：若你生成了 `/workspace/report.docx`，应在最终答复前调用 `report_final_files(files=["/workspace/report.docx"])`。
